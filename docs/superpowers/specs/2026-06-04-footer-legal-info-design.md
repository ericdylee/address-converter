# 푸터 + 개인정보처리방침 설계

- 작성일: 2026-06-04
- 상태: 설계 승인됨 (구현 계획 작성 전)

## Context (왜 / 무엇을)

방문자 신뢰를 높이고 광고(AdSense) 정책을 충족하기 위해, 사이트 하단 푸터에 **저작권·데이터 출처 표기**를 정리하고 **개인정보처리방침** 페이지를 추가한다.

### 운영 형태와 법적 전제 (중요)
- 이 앱은 **개인이 만든 무료 도구이며 수익은 AdSense 광고뿐, 재화·용역 판매 없음**.
- 따라서 **전자상거래법상 "통신판매업자 사업자정보 표시 의무"는 적용되지 않음** (상호·대표자·사업자등록번호·통신판매업 신고번호 등 표시 의무 없음).
- 실제로 의미 있는 것은: ①AdSense용 **개인정보처리방침**(Google 정책상 사실상 필수), ②**저작권·데이터 출처** 표기.
- (법률 자문 아님. 운영 형태가 "판매 있는 사업자"로 바뀌면 사업자정보 표시 의무가 생길 수 있음.)

### 결정된 범위
- **포함:** 개인정보처리방침 페이지 / 저작권 + 데이터 출처 표기.
- **제외(Non-goals):** 이용약관, 면책 문구, 연락처, 전자상거래법 사업자정보.
- **구조:** 공통 `<Footer>` 컴포넌트를 layout에 전역 배치(방식 A) — 기존 페이지별 중복 푸터 제거.

## 현재 상태
- 푸터가 `app/page.tsx`(~93줄), `app/result/page.tsx`(~260줄)에 **인라인으로 중복**됨 (`데이터: 행정안전부 도로명주소 영문 API`).
- 정책/소개 라우트 없음. 라우트는 `/`, `/result` 2개.
- AdSense는 `.env.local`에 `NEXT_PUBLIC_ADSENSE_CLIENT_ID` 미설정(아직 광고 미로드). 방침 문구는 광고가 켜질 때를 대비해 작성.

## 설계

### 1. `components/Footer.tsx` (신규)
- 인터랙션 없는 **서버 컴포넌트**. 3줄:
  1. `데이터: 행정안전부 도로명주소 영문 API` (기존 문구 이전)
  2. `© {연도} 한글 → 영문 주소 변환기` — 연도는 `new Date().getFullYear()`로 자동 갱신
  3. `개인정보처리방침` — `next/link`의 `<Link href="/privacy">`
- 스타일: 기존 톤 유지(`text-xs text-gray-400`, 가운데 정렬, 적당한 패딩). 링크는 hover 시 약간 진하게.

### 2. `app/layout.tsx` 수정
- `import Footer from "@/components/Footer"`.
- `<body>` 안 `{children}` **다음**에 `<Footer />` 배치.
- `body`는 이미 `min-h-full flex flex-col`. 푸터가 항상 화면 하단에 오도록, 페이지 `main`이 남는 공간을 채우게 함(예: Footer에 `mt-auto`, 또는 레이아웃에서 `{children}`을 `flex-1` 래퍼로 감쌈). 기존 페이지 `main`이 `min-h-screen`이므로 콘텐츠가 짧아도 푸터는 자연히 아래에 위치 — 시각 확인으로 마무리.

### 3. 인라인 푸터 제거
- `app/page.tsx`, `app/result/page.tsx`의 `<footer>…</footer>` 블록 삭제(데이터 출처 문구는 공통 Footer로 이전됨).

### 4. `app/privacy/page.tsx` (신규, 개인정보처리방침)
- 한국어 정적 페이지. `metadata.title` 설정. 상단·하단에 `← 홈으로` 링크. 기존 페이지와 동일한 레이아웃 톤(`max-w-2xl`, 카드/여백).
- 문단 구성:
  1. **시행일 / 최종 수정일**: 2026-06-04
  2. **개요**: 본 방침은 '한글 → 영문 주소 변환기'(이하 "서비스")의 개인정보 처리 방식을 설명.
  3. **수집하는 개인정보**: 회원가입·로그인 없음. 이름·연락처 등 개인정보를 직접 수집·저장하지 않음. 입력한 주소 검색어는 영문 변환을 위해 행정안전부 도로명주소 영문 API 조회에만 사용되며 서버에 저장하지 않음.
  4. **쿠키 및 광고(Google AdSense)**: 본 서비스는 Google AdSense로 광고를 게재할 수 있음. Google 등 제3자가 쿠키·광고 식별자로 맞춤형 광고를 제공할 수 있음. 이용자는 Google 광고 설정(adssettings.google.com)에서 맞춤형 광고를 끄거나, aboutads.info에서 제3자 쿠키를 거부할 수 있음. 자세한 내용은 Google 개인정보처리방침 참조.
  5. **외부 링크**: 외부 사이트로 연결되는 링크를 포함할 수 있으며, 외부 사이트의 개인정보 처리에는 책임지지 않음.
  6. **방침 변경**: 변경 시 본 페이지에 게시.

## 파일 변경 요약
- 신규: `components/Footer.tsx`, `app/privacy/page.tsx`
- 수정: `app/layout.tsx`(Footer 삽입), `app/page.tsx`/`app/result/page.tsx`(인라인 푸터 제거)
- 로직·API·타입 변경 없음.

## 검증
1. `npx tsc --noEmit`, `npm run lint` 통과.
2. 개발 서버(단일 인스턴스)에서 `/`, `/result`, `/privacy` 각각 200 응답 + 푸터가 모든 페이지 하단에 표시.
3. 푸터의 "개인정보처리방침" 링크 클릭 → `/privacy` 이동 확인.
4. 모바일 폭(390px)에서 푸터·방침 페이지 레이아웃 깨짐 없음.
5. (선택) 콘텐츠 짧은 페이지에서도 푸터가 화면 하단에 자연스럽게 붙는지 시각 확인.

## 추후(선택)
- 개인정보처리방침에 **문의 이메일** 한 줄 추가하면 완성도↑ (이번엔 연락처 제외 결정).
- AdSense 승인·`NEXT_PUBLIC_ADSENSE_CLIENT_ID` 설정 시 `public/ads.txt`도 채워야 광고 게재됨(별건).
