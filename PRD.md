## 1. 한 줄 요약 (TL;DR)

한글·일본어 주소를 입력하면 **공식 데이터 기반으로 영문 주소를 자동 변환**하고,
해외 사이트 입력 양식에 맞춰 **Street / City / State / Postal Code를 필드별로 따로 복사**할 수 있게 해주는 무료 웹 도구입니다.

---

## 2. 배경 & 문제 정의 (Why)

해외 온라인 서비스(쇼핑몰·계정 가입·구독 서비스 등)에 가입하거나 배송지를 입력할 때, 한국·일본 사용자는 자기 주소를 **영문으로** 적어야 합니다. 여기서 생기는 불편이 이 제품의 출발점입니다.

- **변환 자체가 어렵다** — 도로명/지번 주소를 영문 표기법(로마자)으로 정확히 바꾸기 어렵고, 잘못 적으면 배송·인증이 실패합니다.
- **양식이 칸별로 쪼개져 있다** — 해외 사이트는 보통 `Street Address`, `City`, `State/Province`, `ZIP` 칸이 **따로** 있습니다. 영문 주소 한 줄을 통째로 알아도, 그걸 칸마다 잘라 넣는 일이 번거롭습니다.
- **어느 칸에 뭘 넣어야 할지 헷갈린다** — "구"를 City에 넣어야 하나 State에 넣어야 하나? 서울 같은 광역시는 시(State)와 구(City) 구분이 특히 혼란스럽습니다.

기존 대안(행안부 juso 사이트, 번역기, 수기 작성)은 "한 줄 영문 주소"까지만 주거나 정확도가 낮아, **칸별 입력 + 정확도** 두 가지를 동시에 풀어주지 못합니다.

## 3. 목표와 비목표

### 3.1 제품 목표 (Goals)

1. **정확한 변환** — 공식 데이터(행안부 영문주소 API, 일본우편 공식 로마자 데이터)에 기반해 신뢰할 수 있는 영문 주소를 제공한다.
2. **칸별 복사 UX** — 해외 양식의 칸 구조 그대로 필드를 분리해, 클릭 한 번으로 각 칸을 복사하게 한다.
3. **입력 가이드** — "이 칸엔 보통 이걸 넣으세요"라는 안내로 사용자의 판단 부담을 없앤다.

### 3.2 비즈니스 목표 (핵심)

- **광고 수익화 (AdSense).** 이 제품의 1차 비즈니스 목표는 트래픽을 모아 Google AdSense로 수익을 내는 것입니다.
  - 이 목표 때문에 앱은 **검색(`/`) → 결과(`/result`)** 2-라우트 구조를 택했습니다. 라우트가 전환되는 시점에 AdSense Vignette(전면 광고)가 노출될 기회를 만들기 위함입니다.
  - 즉 "검색 → 결과 이동"이라는 핵심 사용 흐름 자체가 광고 노출 지점과 맞물려 있습니다.

### 3.3 비목표 (Non-goals) — 지금은 안 하는 것

- **회원가입·로그인·계정** — 없음. 누구나 바로 쓰는 익명 도구.
- **주소 저장/동기화 서버** — 없음. 변환은 일회성이며 서버에 사용자 주소를 저장하지 않음.
- **결제·유료 플랜** — 없음. 수익은 광고로만.
- **주소 외 번역(이름·문장 등)** — 범위 밖.

## 4. 타겟 사용자

**주 사용자: 해외 서비스 가입자 전반.** 해외 온라인 서비스에 가입하거나 주소를 입력해야 하는 한국·일본 사용자 누구나가 대상입니다. 특정 직군에 한정하지 않습니다.

대표적인 상황 예시:
- 해외 쇼핑몰(아마존·이베이 등)에서 배송지를 입력하는 사람
- 넷플릭스·페이팔 등 해외 계정 가입 시 청구 주소를 적는 사람
- 유학·이민·해외 서류에 영문 주소가 필요한 사람

**공통점:** "내 한국/일본 주소를 영문 칸에 정확히 넣어야 하는데 방법을 모르겠다"는 일시적·실용적 니즈. 한 번 해결하면 떠나는 **단발성·고빈도 진입**이 특징입니다(= SEO/검색 유입과 광고 모델에 잘 맞음).

## 5. 사용자 시나리오 (User Stories)

1. **한국 주소 변환**
   "해외 쇼핑몰 배송지 칸에 넣을 영문 주소가 필요하다. 한글 주소를 입력하면 후보가 떠서 고르고, 칸별로 복사해 붙여넣고 싶다."

2. **상세주소(동/호) 포함**
   "아파트 동·호수까지 영문으로 넣어야 한다. 검색어에 `101동 502호`를 같이 적으면 알아서 `101-502`로 합쳐주면 좋겠다."

3. **일본 주소 변환**
   "일본 주소를 영문으로 적어야 한다. 우편번호(7자리)를 넣으면 동네·시·도도부현이 채워지고, 번지만 직접 넣으면 일본식 어순으로 합쳐주면 좋겠다."

4. **칸 매칭이 헷갈릴 때**
   "사이트엔 City 칸만 있고 State 칸이 없다. 그럼 뭘 넣어야 하지? — 가이드가 '보통 큰 도시 이름(Seoul)을 넣으세요'라고 알려주면 좋겠다."

## 6. 기능 요구사항 (현재 구현 기준)

### 6.1 한국 주소 변환 (구현 완료 ✅)

- 한글 주소(도로명/지번) 입력 → **300ms 디바운스 자동완성** 드롭다운.
- 행안부 공식 영문주소 API(`business.juso.go.kr`)를 **서버 프록시**(`/api/search-address`)를 통해 호출. (API 키는 서버에만 보관, 브라우저에 노출 안 함)
- 키보드 네비게이션(↑ ↓ Enter Esc), 외부 클릭 시 닫힘, 진행 중 요청 자동 취소(AbortController).
- 검색어에 포함된 **상세주소(동/호) 자동 추출** — 키워드만 API에 질의하고, 동/호는 결과의 Street 칸에 `101-502` 형태로 합침.
- 결과를 **4필드(Street / City / State / Postal Code) + Country**로 분리.

### 6.2 일본 주소 변환 (구현 완료 ✅)

- 홈 상단 **[🇰🇷 한국][🇯🇵 일본] 탭**으로 전환.
- 일본은 무료 자동완성 API가 없어 **우편번호(7자리) 조회** 방식. 일본우편 공식 로마자 데이터(`KEN_ALL_ROME`, 약 12만 건)를 빌드 시 JSON으로 만들어 서버에 내장.
- 사용자가 **번지·건물번호를 직접 입력** → 일본식 어순(번지가 동네 앞: `1-1-1 Marunochi`)으로 합침.
- 동네 없는 우편번호(시 전체) 같은 엣지 케이스 처리.

### 6.3 결과 화면 (구현 완료 ✅)

- **대표 "영문 주소" 한 줄 블록** — 통째로 복사 가능(파란색 강조 카드).
- **필드별 복사 카드** — Street / City / State(또는 Prefecture) / Postal Code / Country 각각 [복사] 버튼.
  - 복사 시 `navigator.clipboard` 우선, 구형 브라우저 대비 `execCommand` 폴백.
  - 복사 완료 시 1.5초간 "복사됨!" + emerald 색 피드백.
- **입력 가이드(접이식)** — 해외 양식 칸 이름 ↔ 우리 필드 매핑표 + 나라별 팁.
- 필수 값 누락 시 "잘못된 접근입니다" 안내 + 검색 페이지 링크.

### 6.4 공통/부가 (구현 완료 ✅)

- 공통 **푸터**(저작권·데이터 출처·개인정보처리방침 링크).
- **개인정보처리방침** 페이지(`/privacy`).
- AdSense 스크립트는 `NEXT_PUBLIC_ADSENSE_CLIENT_ID` 환경변수가 있을 때만 조건부 로드. `public/ads.txt`는 승인 후 채울 자리 준비됨.

## 7. 비기능 요구사항

- **개인정보 최소화** — 사용자 주소를 서버에 저장하지 않음. 변환은 일회성, 값은 URL 파라미터로만 화면 간 전달.
- **API 키 보안** — juso 키는 서버 전용. 키 미설정 시 503 + 한글 안내 메시지(설정 누락을 UI에서 바로 인지).
- **반응형** — 모바일(375px)에서 가로 오버플로우 없이 동작.
- **접근성(현재 부분 충족)** — 탭에 `role="tab"`/`aria-selected`, 가이드 화살표에 `aria-hidden` 등. (개선 여지 있음 → §8)
- **에러 처리** — API 에러는 사용자 친화적 한글 메시지로 변환(개발 모드에서만 원본 detail 노출).

## 8. 디자인 & UX (향후 비중 ↑)

> 사용자가 선택한 **다음 우선순위 방향**입니다. 현재 상태를 먼저 정리하고, 고도화 방향을 제시합니다.

### 8.1 현재 디자인 (구현된 톤)

"세련된 업무 도구(utility tool)" 톤. 화려함보다 **깔끔함·신뢰감** 우선.

- **컬러**
  - 배경: `#f6f8fc` (페이지) / `#f7f8fb` (`globals.css` 토큰), 카드: 흰색
  - 기본 텍스트: `#111827`(gray-900 계열)
  - 주요 액션·강조: **파란색 `blue-600`** (탭 활성, 검색·복사 버튼, 링크)
  - 복사 완료: **emerald-600** (초록)
  - 국가 배지: 한국 `blue`, 일본 `red`
  - 텍스트 선택: 배경 `#dbeafe` / 글자 `#172554`
- **타이포그래피:** Geist Sans(본문) · Geist Mono(영문 주소값 — 칸별 정렬 가독성)
- **형태:** 얕은 그림자 + 섬세한 1px border 중심, 과한 둥근 모서리 지양(`rounded-lg` 수준)
- **레이아웃:** 가운데 정렬 1단(`max-w-3xl`), eyebrow("Address Converter") + 타이틀 + 카드 섹션

### 8.2 디자인 시스템화 (제안)

지금은 색·간격·컴포넌트 스타일이 **여러 파일에 흩어져** 있습니다(`globals.css`, `page.tsx`, `AddressCard.tsx`, `result/page.tsx`). 이를 **디자인 토큰 한 곳**으로 모으는 것이 1순위 과제입니다.

- 색/간격/그림자/라운드 값을 CSS 변수(디자인 토큰)로 통일 → 별도 `DESIGN_SYSTEM.md`로 문서화.
- 반복되는 UI(카드, 버튼, 배지, 입력창)를 **재사용 컴포넌트/클래스**로 정리.

### 8.3 UX 고도화 후보

- **다크모드 토글** (ROADMAP 백로그) — 토큰화가 선행되면 자연스럽게 가능.
- **접근성 강화** — 키보드 포커스 링, 색 대비(WCAG AA), 스크린리더 라벨 점검.
- **복사 피드백 개선** — 토스트/마이크로 인터랙션으로 "복사됨" 인지율 향상.
- **빈/에러 상태 디자인** — "검색 결과 없음", "잘못된 접근"을 더 친절한 빈 상태 화면으로.
- **첫 진입 안내** — 처음 온 사용자가 사용법을 0초에 이해하도록 예시/플레이스홀더 강화.

> ⚠️ **광고 모델과의 균형:** 디자인 개선 시 AdSense 정책(콘텐츠 대비 광고 비중, 오인 클릭 유발 금지)을 해치지 않아야 합니다. 광고 영역과 기능 UI가 시각적으로 명확히 구분되도록 유지합니다.

## 9. 성공 지표 (Metrics)

핵심 목표가 **광고 수익화**이므로, 지표도 트래픽·광고 중심으로 봅니다.

- **유입 (선행 지표)**
  - 월 방문자 수 / 신규 vs 재방문
  - 검색 유입(SEO) 비중 — "영문 주소 변환" 등 키워드 노출/클릭
- **참여 (제품 건강도)**
  - 검색 → 결과(`/result`) 도달률 (= 핵심 흐름 완주율)
  - 필드별 [복사] 클릭률 (= 실제 가치 전달 신호)
  - 이탈률 / 세션당 변환 횟수
- **수익 (결과 지표)**
  - AdSense 노출수·CTR·RPM, 페이지뷰당 수익
  - Vignette 노출 빈도(라우트 전환당)

> 현재 분석 도구는 미정. **Open Question:** GA4 / Vercel Analytics 등 측정 수단 도입 필요.

상세 구조는 `CLAUDE.md` 참조. 요약만 적습니다.
